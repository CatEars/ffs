Debug: Megaphone Template Binds (Issue #35)

Issue #35

Bind markers ($attr="stateKey") inside <template> elements were not registered when the template was cloned and rendered via megaphone.template() + megaphone.renderEach(), because #bindValuesAndEventListeners() only ran once at init() time, before the cloned elements existed in the DOM.

Each test below renders rows from a template that contains bind markers. Clicking the toggle button changes the relevant state; if the fix is working the rendered rows update immediately.

Test 1 — $class bind in rendered template rows

Each row is cloned from a <template> whose root <div> carries $class="rowClass". Clicking Toggle highlight alternates the class between item-row and item-row highlighted. All rows should visually change at once.

highlight: off

Test 2 — multiple bind markers in one template

Each row is cloned from a template containing both $class="dotClass" (status dot) and $textcontent="statusText" (label text). Clicking Toggle online/offline should simultaneously update both the dot colour and the label on every row.
