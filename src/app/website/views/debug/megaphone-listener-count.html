Debug: Megaphone Listener Count

Listener Count Stability Test

Each time renderEach (or render / renderIf) re-renders a list, old DOM elements are removed and new ones are created. Without cleanup, the internal boundValues map accumulates stale references to removed elements on every render, causing unbounded memory growth.

Click Few items and Many items repeatedly. The Listener Counts table must show counts that match the current number of rendered rows — never growing beyond that — proving that stale entries are pruned on each re-render. megaphone.debugger.listenerCount() is called after each render to refresh the display.
